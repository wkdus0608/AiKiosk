## 1. 전역 레이아웃 및 공통 요소

> **적용 지식** : 유니버셜 디자인 원칙(공평한 사용, 인지할 수 있는 정보) 및 접근성(글자 크기 조절)

* **[개선안] 상시 노출형 접근성 도구 막대(Accessibility Toolbar) 추가**

  * **현황** : 현재 헤더는 타이틀(`Interactive Kiosk`)과 PWA 설치 버튼만 존재하며, 저시력자나 고령자를 지원하는 인터페이스가 부재합니다.
  * **변경 지점** : 헤더 영역 또는 그 바로 아래에 **상시 노출되는 '쉬운 화면 설정 도구 막대'**를 추가합니다.

  1. **글자 크기 조절** : 보통 / 크게 / 아주 크게 (3단계) 조절 버튼 제공
     * CSS 변수(예: `--font-scale: 1.0 / 1.3 / 1.6`)를 정의하여 폰트 크기가 일괄 유동적으로 변경되도록 설계합니다.
  2. **고대비 모드 토글** : 검은색 배경에 노란색/흰색 텍스트 조합을 사용하는 고대비 테마(교통약자 표준 색상) 전환 스위치 제공
     * 토글 시 `body` 태그에 `.high-contrast` 클래스를 주입하여 주요 컴포넌트의 배경과 전경색 대비를 극대화합니다.
  3. **음성 안내(TTS) On/Off** : 화면 요소 및 안내 스크립트를 음성으로도 함께 송출해 인식을 보완합니다.


## 2. 시작 화면 

> **적용 지식** : 단순하고 직관적인 사용, 실수에 대한 관용, 디지털 소외 방지

시작화면에는 3개의 버튼만 존재해야 함. '이어폰 감지 활성화', '도움 요청', '시작하기'

## 3. 일반 주문 화면

> **적용 지식** : 힉의 법칙(Hick's Law), 적은 신체적 노력, 접근과 사용 공간

* **[개선안 1] 카테고리 탭 도입을 통한 힉의 법칙(인지 부하 감소) 적용**
  * **현황** : 등록된 모든 메뉴(`stockList`)가 단일 화면에 그리드/리스트로 무작위 나열되고 있습니다. 메뉴 개수가 많아질수록 고령 사용자의 의사결정 시간이 기하급수적으로 증가합니다.
  * **변경 지점** : 화면 최상단에 **"커피 / 음료 / 디저트" 등 직관적인 대분류 카테고리 탭**을 제공하여 한 번에 보여지는 선택지의 개수를 제한하고 인지 부하를 줄입니다.
* **[개선안 2] 장바구니 수량 조절 버튼(`+`, `-`) 크기 확대**
  * **현황** : 장바구니 리스트의 수량 조절 버튼이 `circle dense` 스타일로 설정되어 터치 영역이 지나치게 좁습니다. 관절염이 있거나 손떨림이 있는 고령자는 오동작을 일으키기 쉽습니다.
  * **변경 지점** : 버튼 크기를 최소 가로세로 48px 이상(터치 타겟 표준 규격)으로 확대하고, 버튼 간 간격을 넓혀 **적은 신체적 노력**으로도 정확히 누를 수 있도록 변경합니다.
* **[개선안 3] 장바구니 영역 상시 노출 및 시각화**
  * **현황** : 웹(Non-Electron) 모드에서는 장바구니가 숨겨져 있다가 토글해야 열립니다. 내가 고른 상품이 제대로 담겼는지 인지하기 어렵습니다.
  * **변경 지점** : 상품을 담을 때 장바구니 영역이 화면 하단에 밝은 색상으로 상시 유지되거나, 담기 완료 시 장바구니가 흔들리는 마이크로 애니메이션 피드백을 주어 성공적으로 담겼음을 시각화합니다.



## 4. 음성 주문 화면 (![](vscode-file://vscode-app/Applications/Antigravity%20IDE.app/Contents/Resources/app/extensions/theme-symbols/src/icons/files/document.svg)

VoiceOrder.vue)

> **적용 지식** : 실수에 대한 관용, 교통약자 이동성 향상(사용 편의성), 인지할 수 있는 정보

* **[개선안 1] 키보드 Space(PTT) 외 스크린 터치 마이크 버튼 지원**
  * **현황** : 현재 음성 녹음 제어(`activatePTT`, `deactivatePTT`)가 키보드의 `Space` 키에 매핑되어 작동합니다. 실제 키오스크 환경에서는 키보드가 없으므로 터치 스크린만으로 제어할 수 있어야 합니다.
  * **변경 지점** : 화면 중앙의 큰 마이크 인디케이터 자체를 **"터치하여 말하기 / 터치 완료"** 기능을 수행하는 대형 버튼으로 개편합니다.
* **[개선안 2] 대화형 음성 인식 상태 안내 텍스트 강화**
  * **현황** : 인식 성공/실패 여부를 텍스트 스크립트 한 줄과 오디오 사운드에만 의존하고 있습니다.
  * **변경 지점** : 마이크 인디케이터 내부나 주변에 **진행 상태(예: 🎤 "듣고 있어요. 메뉴를 말씀해 주세요", 🔄 "음성을 분석하고 있어요", ❌ "죄송해요, 다시 한번 말씀해 주세요")**를 애니메이션과 함께 직관적인 한글로 상세히 표기하여 고령자의 심리적 불안감과 디지털 격차를 해소합니다.
